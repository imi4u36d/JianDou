# 任务创建四角色三轮迭代

## 目标

围绕任务创建链路，协同优化三类核心对象：

- 创建参数（CreateTaskRequest）
- 风格预设（TaskPreset）
- 新建页 UI 交互（/tasks/new）

## 角色分工

- 资深产品设计师：收敛参数暴露层级，定义模式边界和建单路径。
- 资深剪辑师：定义短剧/混剪在素材组织、题材和风格上的可执行规则。
- 开发：落实前后端字段对齐、交互联动和提交校验。
- 测试：定义创建前可判定的校验项并验证回归风险。

## V1（参数与预设模型对齐）

### 产品设计师

- 结论：预设必须显式带 `editingMode`，否则用户不知道预设适配短剧还是混剪。

### 剪辑师

- 结论：混剪预设应带题材和风格默认值，避免进入混剪后仍需二次补参。

### 开发落地

- 前端 `TaskPreset` 增加：`editingMode`、`mixcutContentType`、`mixcutStylePreset`。
- 新建页预设卡片显示模式标签（短剧/混剪）。
- 新增旅行分镜混剪回退预设。

### 测试关注

- 校验 `GET /presets` 返回字段可被前端完整消费。
- 校验模式标签展示与预设实际值一致。

## V2（剪辑规则与模式联动）

### 产品设计师

- 结论：建单规则应在前端即时生效，不应把模式冲突留给后端报错。

### 剪辑师

- 结论：
  - 短剧：只允许单主素材。
  - 混剪：至少两条素材才能稳定形成分镜编排。

### 开发落地

- 新建页文案改为：短剧模式仅支持 1 条主视频。
- 在短剧模式追加多素材时自动切换到混剪模式。
- 从历史任务复制时按 `editingMode` 匹配预设。

### 测试关注

- 场景 1：短剧模式下多选素材，是否自动切换混剪。
- 场景 2：切回短剧模式时是否保留主素材并移除多余素材。

## V3（提交前校验与可见性）

### 产品设计师

- 结论：用户需要在提交前知道“是否可提交”以及具体未通过项。

### 剪辑师

- 结论：素材数量和时长区间是成片稳定性的硬门槛，必须显式提示。

### 开发落地

- 新建页新增“提交校验”区块。
- 校验项：标题、素材数量、时长区间、产出数量。
- 提交按钮与校验联动：未通过时禁用，并提示首个失败项。

### 测试关注

- 场景 1：任一校验失败时按钮禁用。
- 场景 2：全部通过时可正常上传并创建任务。
- 场景 3：回归验证 AI 自动生成提示词、复制任务、预设切换流程。
