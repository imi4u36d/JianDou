# UI / UX 设计说明

本文档对应当前 `apps/web` 前端实现，范围仅包含已经落地的页面、字段、状态和交互，不描述未实现能力。

## 1. 页面结构

当前前端是一个三页工作流，路由由 `Vue Router` 管理：

- `/tasks`：任务列表页，查看历史任务和当前任务进度。
- `/tasks/new`：新建任务页，上传原始视频并填写剪辑参数。
- `/tasks/:id`：任务详情页，查看任务状态、阶段进度和生成结果。

全局结构由顶部导航和主内容区组成，顶部提供 `任务列表` 和 `新建任务` 两个入口。整体布局以单任务流为主，符合“上传 -> 配置 -> 创建 -> 查看结果”的操作顺序。

后台管理路由（`/admin`）采用常规管理系统布局，包含：

- `/admin`：运营总览（指标、失败任务、日志）
- `/admin/tasks`：任务管理表格（筛选、批量操作、详情）
- `/admin/tasks/:id`：任务诊断详情
- `/admin/system`：系统配置与日志筛选

## 2. 核心交互

### 2.1 任务列表页

- 默认加载任务列表，首屏展示加载态。
- 列表为空时显示空态，引导用户进入新建任务。
- 任务卡片展示任务标题、平台、素材数量、进度和更新时间。
- 点击任务卡片进入详情页。
- 页面每 4 秒轮询刷新一次列表，适合查看处理中的任务进度变化。

### 2.2 新建任务页

- 用户先选择剪辑模式（短剧 / 混剪），再选择素材并填写任务参数。
- 预设按当前模式过滤展示，避免短剧参数误套到混剪任务（反之亦然）。
- 提交动作是“先上传文件，再创建任务”：
  - `uploadVideo(file)` 上传原视频。
  - `createTask(payload)` 发送任务配置。
- 短剧模式仅支持 1 条素材；混剪模式至少 2 条素材。
- 在短剧模式下追加多素材会自动切换到混剪模式。
- 提交前会显示参数校验清单（标题、素材数量、时长区间、产出数量）。
- 如果最小时长大于最大时长，页面会拦截提交并提示参数错误。
- 提交成功后自动跳转到任务详情页。
- 提交中按钮会显示“提交中...”，避免重复点击。

### 2.3 任务详情页

- 左侧展示任务进度、状态徽标、阶段时间线和任务配置摘要。
- 右侧展示生成结果列表，每条结果包含视频预览、推荐理由、起止时间和下载入口。
- 如果任务失败，左侧展示错误信息，并提供 `失败重试` 按钮。
- 页面每 3 秒轮询刷新任务详情，适合异步渲染场景。

## 3. 状态设计

当前系统的任务状态由后端返回，前端仅做展示和弱交互控制：

- `PENDING`：待处理。
- `ANALYZING`：素材分析中。
- `PLANNING`：剪辑规划中。
- `RENDERING`：视频渲染中。
- `COMPLETED`：任务完成。
- `FAILED`：任务失败。

### 3.1 列表状态

- 加载中：显示“正在加载任务列表...”。
- 空态：显示“还没有任务”，引导创建。
- 正常态：展示任务卡片、状态徽标和进度条。

### 3.2 详情状态

- 加载中：显示“正在加载任务详情...”。
- 处理中：时间线中按状态点亮阶段。
- 失败：显示错误消息和重试入口。
- 完成：展示输出列表，允许逐条预览和下载。

### 3.3 输出状态

- 无输出：显示占位提示，说明完成后会在此出现结果。
- 有输出：按输出条目渲染视频播放器、推荐理由和下载链接。

## 4. 表单字段

当前新建任务页使用的字段与类型如下：

| 字段 | 类型 | 当前实现 |
| --- | --- | --- |
| `title` | 文本 | 必填，任务标题 |
| `sourceAssetId` | 文本 | 上传成功后回填 |
| `sourceAssetIds` | 文本数组 | 上传后回填，多素材模式使用 |
| `sourceFileName` | 文本 | 上传文件名回填 |
| `sourceFileNames` | 文本数组 | 上传后回填，多素材模式使用 |
| `editingMode` | 枚举文本 | `drama` / `mixcut` |
| `mixcutEnabled` | 布尔 | 由 `editingMode` 自动同步 |
| `mixcutContentType` | 枚举文本 | `generic` / `travel` / `drama`（混剪模式） |
| `mixcutStylePreset` | 枚举文本 | 题材对应的混剪风格（混剪模式） |
| `platform` | 枚举文本 | `douyin` / `kuaishou` / `xiaohongshu` / `wechat` |
| `aspectRatio` | 枚举文本 | `9:16` / `16:9` |
| `minDurationSeconds` | 数值 | 默认 `15`，最小 `5`，最大 `120` |
| `maxDurationSeconds` | 数值 | 默认 `30`，最小 `5`，最大 `120` |
| `outputCount` | 数值 | 默认 `3`，最小 `1`，最大 `10` |
| `introTemplate` | 枚举文本 | `none` / `cold_open` / `flash_hook` / `pressure_build` |
| `outroTemplate` | 枚举文本 | `none` / `suspense_hold` / `follow_hook` / `question_freeze` |
| `creativePrompt` | 多行文本 | 可选，补充剪辑偏好 |
| `transcriptText` | 多行文本 | 可选，支持粘贴或导入字幕/台词 |

注意：

- 当前页面已实现模式和素材数量的联动校验。
- `platform` 的文案与内部值一致，后续如要做运营侧友好显示，需要后端或字典层统一映射。

## 5. 任务流

当前实际任务流是单向的异步链路：

1. 用户在新建任务页选择模式和预设。
2. 用户上传 1 条（短剧）或多条（混剪）原始视频。
3. 用户填写时长、模板、提示词和字幕语义输入。
4. 前端先做创建前校验清单。
5. 前端上传素材并获得 `assetId` 列表。
6. 前端将 `sourceAssetId/sourceAssetIds` 与任务参数一起创建任务。
7. 任务进入后端处理流程。
8. 任务列表页和详情页通过轮询刷新进度。
9. 任务完成后，详情页展示多个输出素材。
10. 用户可直接预览或下载每个素材。
11. 任务失败时，用户可在详情页重试。

当前前端没有做队列视图、人工拆片编辑、批量确认面板，也没有 WebSocket 推送，全部依赖轮询刷新。

## 6. 面向运营投放人员的使用重点

- 操作路径要短：先上传，再填参数，再等结果，避免多层配置。
- 参数要可控：当前只暴露平台、比例、时长区间、数量、片头、片尾和创意补充，足够支持常见投放需求。
- 结果要可直接使用：生成完成后直接提供预览和下载，不要求二次导出流程。
- 状态要可理解：通过状态徽标、进度条和阶段时间线，让运营知道当前卡在哪一步。
- 异常要可恢复：失败页提供重试，减少重新上传和重填的成本。
- 适合高频批量操作：列表页聚合历史任务，详情页聚焦单个任务结果，符合运营反复试投的工作方式。

## 7. 设计边界

以下内容当前未在前端实现，文档不作为承诺范围：

- 片段级拖拽裁切。
- 素材批量打标签和批量审批。
- 版本对比和 A/B 结果管理。
- 片头片尾的可视化时间轴编辑。
- 实时流式推送状态。

如果后续要扩展，本页面应优先在任务详情页增加结果编辑区，而不是改变当前三页结构。
